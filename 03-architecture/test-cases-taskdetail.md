## 1. 流程可视化测试 (Phase 1)

### TC-001: Stage1 流程展示
**前置条件**: 任务处于 planning 或 researching 状态
**测试步骤**:
1. 打开任务详情页
2. 查看流程可视化区域
**预期结果**:
- [ ] Stage1 显示5个步骤: RSS聚合、质量评估、热点分析、竞品分析、评分排序
- [ ] 已完成的步骤显示绿色 ✓ 标记
- [ ] 进行中的步骤显示动画效果

### TC-002: Stage2 流程展示
**前置条件**: 任务处于 researching 或 writing 状态
**测试步骤**:
1. 打开任务详情页
2. 查看 Stage2 深度研究流程
**预期结果**:
- [ ] 显示4个步骤: 数据采集、清洗、分析、洞察
- [ ] 流程状态根据任务状态正确显示

### TC-003: 流程状态指示器
**测试步骤**:
1. 创建不同状态的任务
2. 观察流程可视化
**预期结果**:
- [ ] 已完成阶段显示绿色边框
- [ ] 当前阶段显示橙色边框和动画
- [ ] 未开始阶段显示灰色和较低透明度

## 2. 蓝军评审界面测试 (Phase 2)

### TC-101: 评审统计概览卡片
**前置条件**: 任务有蓝军评审数据
**测试步骤**:
1. 切换到"蓝军评审" Tab
2. 查看统计概览区域
**预期结果**:
- [ ] 显示4个统计卡片: 严重问题(红)、改进建议(黄)、亮点(绿)、总数(紫)
- [ ] 数字正确反映评审数据

### TC-102: 处理进度条
**测试步骤**:
1. 查看处理进度区域
**预期结果**:
- [ ] 进度条分段显示: 已接受(绿)、已忽略(灰)、待处理(黄)
- [ ] 显示已处理/总数统计
- [ ] 图例正确显示各状态数量

### TC-103: 专家评审分工展示
**测试步骤**:
1. 查看专家评审分工区域
**预期结果**:
- [ ] 显示4个专家角色卡片: 事实核查员、逻辑检察官、行业专家、读者代表
- [ ] 每个卡片显示图标、名称、职责描述
- [ ] 鼠标悬停显示重新评审按钮

### TC-104: 评审项决策按钮
**前置条件**: 任务状态为 awaiting_approval
**测试步骤**:
1. 找到待处理的评审意见
2. 点击"接受修改"按钮
**预期结果**:
- [ ] 按钮点击后状态变为"已接受"
- [ ] 评审项透明度降低
- [ ] 统计数字更新

### TC-105: 批量决策功能
**前置条件**: 任务状态为 awaiting_approval，有多条待处理意见
**测试步骤**:
1. 点击"全部接受"按钮
2. 确认弹窗
**预期结果**:
- [ ] 所有待处理意见状态变为"已接受"
- [ ] 进度条更新为100%

### TC-106: 严重问题强制校验
**前置条件**: 任务有待处理的严重问题
**测试步骤**:
1. 尝试点击"确认发布"按钮
**预期结果**:
- [ ] 弹出警告: "有 X 个严重问题未处理，处理后才能进入确认环节"
- [ ] 无法进入确认流程

### TC-107: 重新评审功能
**前置条件**: 任务状态为 awaiting_approval
**测试步骤**:
1. 鼠标悬停专家卡片
2. 点击重新评审按钮
3. 确认弹窗
**预期结果**:
- [ ] 显示重新评审成功提示
- [ ] 该专家的评审意见更新
