## 1. 文档概述

### 1.1 背景

内容生产流水线是一套 AI 驱动的产业研究报告生产系统，实现从选题策划到终稿发布的全流程自动化。当前系统已覆盖选题、研究、写作、评审四大阶段，支持 BlueTeam 专家级评审。

### 1.2 目标

构建完整的内容生产流水线，覆盖以下核心能力：
- **智能选题**: 基于 RSS 聚合、热点分析、竞品监控的自动化选题
- **深度研究**: 数据采集、清洗、分析与洞察提炼
- **文稿生成**: AI 辅助初稿生成、润色、事实核查
- **质量评审**: 专家评审、读者测试、逻辑检查

### 1.3 成功标准

| 指标 | 现状 | 目标 | 验证方式 |
|------|------|------|---------|
| 选题响应时间 | 人工发现 2-3 天 | < 1 小时自动发现 | 热点推送延迟 |
| 研究效率 | 2 天/篇 | < 0.5 天/篇 | 计时统计 |
| 内容质量分 | 70 分 | > 85 分 | 专家评审 |
| 事实核查准确率 | 人工抽查 | > 95% 自动核查 | 抽检验证 |
| 读者满意度 | - | > 80% | 问卷调研 |

## 3. 功能规格

### 3.1 选题策划 (Topic Planning)

#### 3.1.1 RSS 聚合 (RSS Aggregator)

**功能描述**:
- 支持多源 RSS 订阅源管理
- 自动抓取、去重、分类
- 定时轮询（15 分钟）

**技术规格**:
```yaml
抓取频率:
  P0 源(官方公告): 实时
  P1 源(头部财经): 15 分钟
  P2 源(垂直媒体): 30 分钟

存储字段:
  - id: 唯一标识
  - title: 标题
  - source: 来源
  - sourceUrl: 原文链接
  - summary: 摘要
  - publishedAt: 发布时间
  - hotScore: 热度分
  - tags: 标签
  - sentiment: 情感倾向
```

**接口**:
- `GET /api/v1/quality/rss-sources` - 获取 RSS 源列表
- `POST /api/v1/quality/rss-sources` - 添加 RSS 源
- `POST /api/v1/quality/rss-sources/crawl` - 触发抓取
- `GET /api/v1/quality/items` - 获取文章列表

#### 3.1.2 质量评估 (Quality Assessment)

**功能描述**:
- 数据源可信度评分
- 内容质量预筛选
- 虚假信息检测

**评估维度**:
| 维度 | 权重 | 说明 |
|------|------|------|
| 来源权威性 | 30% | 官方 > 头部媒体 > 垂直媒体 |
| 内容完整度 | 25% | 结构、数据、引用完整性 |
| 时效性 | 20% | 发布时间距离当前时间 |
| 传播度 | 15% | 转发、阅读、互动数据 |
| 情感客观性 | 10% | 情感中立程度 |

**输出**:
```json
{
  "qualityScore": 85,
  "credibilityLevel": "high",
  "recommendation": "recommend",
  "riskFlags": []
}
```

#### 3.1.3 热点分析 (Hot Topic Analysis)

**功能描述**:
- 热点发现与追踪
- 趋势预测
- 热度评分与排序

**热点算法**:
```
热点分 = 传播速度 × 0.3 + 情感强度 × 0.25 + 权威来源占比 × 0.25 + 时效性 × 0.2

趋势判断:
- 上升: 连续 3 个时间点热度增长 > 20%
- 平稳: 波动范围 < 10%
- 下降: 连续 3 个时间点热度下降 > 20%
```

**界面原型**:
```
┌─────────────────────────────────────────────────────────────┐
│  🔥 热点追踪                                                  │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  趋势筛选: [全部] [🔥 上升] [➡️ 平稳] [📉 下降]             │
│                                                             │
│  ┌───────────────────────────────────────────────────────┐ │
│  │ 📈 新能源汽车销量超预期 (热度: 95)  趋势: 🔥 上升     │ │
│  │ ────────────────────────────────────────────────────  │ │
│  │ 来源: 财新、第一财经、36氪 (15 篇报道)                 │ │
│  │ 情绪: 积极 😊  |  相关研报: 3 篇                       │ │
│  │ [查看详情] [一键立项]                                 │ │
│  └───────────────────────────────────────────────────────┘ │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

#### 3.1.4 竞品分析 (Competitor Analysis)

**功能描述**:
- 竞品账号监控
- 内容差异化分析
- 空白切入点发现

**分析维度**:
- 竞品选题覆盖度
- 发布时间分布
- 内容角度分析
- 互动数据对比

**输出**:
```json
{
  "competitorCoverage": 75,
  "differentiationScore": 82,
  "gaps": [
    "缺少政策解读角度",
    "缺少国际市场对比"
  ],
  "recommendations": [
    "建议从供应链角度切入",
    "可结合 ESG 话题"
  ]
}
```

#### 3.1.5 评分排序 (Scoring & Ranking)

**功能描述**:
- 多维度智能排序
- 优先级推荐
- 选题决策支持

**排序因子**:
```
综合得分 = 热点分 × 0.3 + 质量分 × 0.25 + 差异化分 × 0.2 + 时效分 × 0.15 + 资源匹配度 × 0.1
```

### 3.2 深度研究 (Deep Research)

#### 3.2.1 数据采集 (Data Collection)

**功能描述**:
- 多源数据自动采集
- API 对接
- 定向爬虫

**数据源**:
| 类型 | 来源 | 采集方式 |
|------|------|---------|
| 行业数据 | Wind、Bloomberg | API |
| 政策文件 | 政府网站、监管公告 | RSS + 爬虫 |
| 研报资料 | 券商研报、咨询报告 | PDF 解析 |
| 新闻资讯 | 财经媒体、垂直媒体 | RSS |
| 社交媒体 | Twitter、微博 | API |

#### 3.2.2 数据清洗 (Data Cleaning)

**功能描述**:
- 去重处理
- 格式标准化
- 异常值检测与处理

**清洗规则**:
```yaml
去重规则:
  - 标题相似度 > 90% 视为重复
  - URL 完全相同的去重
  - 内容 MD5 相同的去重

格式标准化:
  - 日期格式统一: YYYY-MM-DD
  - 货币单位统一: 万元人民币
  - 数字精度: 保留 2 位小数

异常值检测:
  - 超出 3 个标准差的数据标记
  - 缺失值填充或删除
```

#### 3.2.3 数据分析 (Data Analysis)

**功能描述**:
- 统计分析
- 趋势分析
- 关联分析

**分析方法**:
| 分析类型 | 方法 | 输出 |
|---------|------|------|
| 描述统计 | 均值、中位数、标准差 | 数据概览 |
| 趋势分析 | 时间序列、移动平均 | 趋势图 |
| 对比分析 | 同比、环比、对标 | 对比表 |
| 关联分析 | 相关系数、回归分析 | 关联图 |
| 聚类分析 | K-means、层次聚类 | 分类标签 |

#### 3.2.4 洞察提炼 (Insight Extraction)

**功能描述**:
- AI 辅助洞察生成
- 核心观点提炼
- 结论总结

**洞察类型**:
```
🔍 发现型洞察: 从数据中发现新模式、新趋势
⚠️ 风险型洞察: 识别潜在风险和警示信号
💡 机会型洞察: 发现市场机会和投资价值
📊 验证型洞察: 验证或证伪既有假设
```

### 3.3 文稿生成 (Content Generation)

#### 3.3.2 内容润色 (Content Polishing)

**功能描述**:
- 语言风格优化
- 可读性提升
- 专业术语校准

**润色维度**:
| 维度 | 优化目标 | 技术方案 |
|------|---------|---------|
| 语言流畅度 | 消除生硬表达 | LLM 重写 |
| 专业度 | 术语准确、表达规范 | 术语库匹配 |
| 可读性 | Flesch 阅读 ease > 60 | 句式简化 |
| 一致性 | 风格统一 | 风格模板 |

#### 3.3.3 事实核查 (Fact Checking)

**功能描述**:
- 数据点自动提取
- 多源交叉验证
- 可信度评分

**核查流程**:
```
1. 提取可验证声明
   └── 识别: 数字、日期、人名、机构名
   
2. 多源验证
   └── 搜索: 至少 2 个独立来源
   └── 比对: 确认一致性
   
3. 可信度评级
   ├── ✅ 已验证: 多源一致
   ├── ⚠️ 存疑: 单源或部分一致
   └── ❌ 错误: 多源矛盾
```

#### 3.3.4 格式调整 (Formatting)

**功能描述**:
- 排版优化
- 图表自动插入
- 格式标准化

**格式模板**:

## 4. 数据模型

### 4.1 核心实体

#### Topic (选题)
```typescript
interface Topic {
  id: string;
  title: string;
  source: 'rss' | 'hot_topic' | 'competitor' | 'manual';
  sourceId?: string;
  hotScore: number;
  qualityScore: number;
  differentiationScore: number;
  status: 'pending' | 'approved' | 'rejected' | 'in_progress';
  priority: 'high' | 'medium' | 'low';
  tags: string[];
  recommendedAngles: string[];
  relatedReports: string[];
  createdAt: string;
  updatedAt: string;
}
```

#### ResearchData (研究数据)
```typescript
interface ResearchData {
  id: string;
  topicId: string;
  dataType: 'raw' | 'cleaned' | 'analyzed' | 'insight';
  source: string;
  content: any;
  metadata: {
    collectionTime: string;
    cleaningRules: string[];
    analysisMethod: string;
    confidence: number;
  };
  tags: string[];
  createdAt: string;
}
```

#### Draft (文稿)
```typescript
interface Draft {
  id: string;
  topicId: string;
  version: number;
  status: 'draft' | 'polishing' | 'checking' | 'reviewing' | 'final';
  outline: OutlineNode[];
  content: string;
  wordCount: number;
  readabilityScore: number;
  factCheckResult: FactCheckResult;
  reviewRounds: ReviewRound[];

  updatedAt: string;
}
```

```typescript

  id: string;
```

## 8. 项目规划

### 8.1 里程碑

| 阶段 | 时间 | 目标 | 交付物 |
|------|------|------|--------|
| M1 | 第 1-2 周 | 选题策划完善 | RSS、热点、竞品功能完整 |
| M2 | 第 3-4 周 | 深度研究上线 | 数据采集、分析、洞察功能 |
| M3 | 第 5-6 周 | 文稿生成上线 | 初稿、润色、格式功能 |
| M4 | 第 7-8 周 | 质量评审上线 | 事实核查、专家评审功能 |
| M5 | 第 9-10 周 | 全流程打通 | 端到端自动化生产 |

### 8.2 验收标准

**阶段 1 验收**:
- [ ] RSS 源管理功能完整
- [ ] 热点发现准确率 > 80%
- [ ] 竞品分析覆盖度 > 70%

**阶段 2 验收**:
- [ ] 数据采集成功率 > 95%
- [ ] 数据清洗自动化率 > 90%
- [ ] 洞察生成可用率 > 80%

**阶段 3 验收**:
- [ ] 初稿生成质量分 > 75
- [ ] 润色后可读性提升 > 20%
- [ ] 事实核查覆盖率 100%

**阶段 4 验收**:
- [ ] BlueTeam 评审可用
- [ ] 专家评审评分一致性 > 80%
- [ ] 读者测试满意度 > 80%
