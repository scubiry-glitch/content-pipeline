## 1. 需求价值（承接 WHY.md）

### 1.1 与 WHY.md 的关联

根据 [WHY.md](../WHY.md)，项目核心痛点是**242份研报无法高效复用**，导致：
1. 没有知识积累
2. 产出效率极低
3. 无法规模化

**v3.0 解决的关键痛点**:

| WHY.md 痛点 | v3.0 解决方案 | 价值量化 |
|------------|--------------|---------|
| "242份研报无法高效复用" | 研报自动解析 + 结构化存储 | 研报复用率从 **5% → 60%** |
| "产出效率极低" | 智能素材库 + 标签检索 | 资料查找时间减少 **50%** |
| "无法规模化" | 热点追踪自动化 | 热点响应从 **天级 → 小时级** |
| 内容质量把控 | 质量评分体系 | 素材质量提升 **30%** |

### 1.2 核心场景

**场景1：研报智能入库**
```
用户上传PDF研报 → AI自动解析
→ 提取：标题、作者、机构、摘要、核心观点
→ 生成：标签、质量评分
→ 存储：结构化入库，可检索复用
```

**场景2：素材智能检索**
```
用户搜索"新能源汽车政策"
→ 系统返回：
  - 相关研报3篇（按质量排序）
  - 相关热点2个
  - 历史素材5个
→ 一键引用到当前研究
```

**场景3：热点自动追踪**
```
系统每小时抓取RSS源
→ 提取热点话题
→ 与在研内容匹配
→ 推送相关热点提醒
```

### 1.3 成功标准

| 指标 | 现状 | 目标 | 验证方式 |
|------|------|------|---------|
| 研报解析成功率 | 人工处理 | >90%自动解析 | 解析日志 |
| 研报复用率 | <5% | >60% | 引用统计 |
| 素材检索效率 | 10分钟/次 | <1分钟/次 | 操作计时 |
| 热点发现时间 | 手动1天 | 自动1小时 | 推送延迟 |

## 2. 功能清单

### 2.1 研报解析 (Report Parser)

**核心功能**:
- 📄 PDF/Word 研报上传
- 🤖 AI自动解析内容结构
- 🏷️ 自动提取标签和关键词
- ⭐ 质量评分（0-100）

**解析字段**:
| 字段 | 说明 | 提取方式 |
|------|------|---------|
| title | 研报标题 | AI提取 |
| authors | 作者列表 | AI提取 |
| institution | 发布机构 | AI提取 |
| publishDate | 发布日期 | AI提取 |
| abstract | 摘要 | AI提取 |
| keyPoints | 核心观点（3-5条） | AI生成 |
| tags | 标签 | AI生成 |
| qualityScore | 质量评分 | AI评估 |

**质量评分维度**:
```
authority:      机构权威性（30%）
completeness:   内容完整度（25%）
freshness:      时效性（25%）
originality:    原创度（20%）
```

### 2.2 热点追踪 (Hot Topic Tracker)

**核心功能**:
- 🌐 RSS源自动抓取
- 🔥 热点话题提取
- 📊 热度趋势分析
- 🔔 相关热点推送

**追踪来源**:
- 财经媒体RSS（财新、第一财经等）
- 行业资讯源
- 社交媒体趋势

**热点字段**:
| 字段 | 说明 |
|------|------|
| title | 热点标题 |
| source | 来源 |
| publishTime | 发布时间 |
| hotScore | 热度分数 |
| sentiment | 情感倾向 |
| relatedTopics | 相关话题 |

### 2.3 素材库 (Asset Library)

**核心功能**:
- 📁 素材分类管理
- 🔍 标签检索
- 💾 素材引用
- 📈 使用统计

**素材类型**:
- 研报（解析后的结构化数据）
- 数据图表
- 引用观点
- 背景资料

### 4.2 核心模块设计

```typescript
// 研报服务
class ReportService {
  // 上传并解析研报
  async uploadAndParse(file: File): Promise<Report>;

  // 获取研报列表
  async listReports(filters: ReportFilters): Promise<Report[]>;

  // 获取研报详情
  async getReport(id: string): Promise<Report>;

  // 搜索研报
  async searchReports(query: string): Promise<Report[]>;

  // 获取智能推荐
  async getRecommendations(context: string): Promise<Report[]>;
}

// RSS热点服务
class RSSCollector {
  // 抓取RSS源
  async fetchRSS(sources: RSSSource[]): Promise<RSSEntry[]>;

  // 提取热点
  async extractHotTopics(entries: RSSEntry[]): Promise<HotTopic[]>;

  // 分析趋势
  async analyzeTrends(topic: string): Promise<TrendAnalysis>;
}

// 素材库服务
class AssetService {
  // 添加素材
  async addAsset(asset: CreateAssetData): Promise<Asset>;

  // 标签检索
  async searchByTags(tags: string[]): Promise<Asset[]>;

  // 全文检索
  async fullTextSearch(query: string): Promise<Asset[]>;

  // 引用素材
  async quoteAsset(id: string, targetId: string): Promise<void>;
}
```

## 6. 数据库 Schema

```sql
-- 研报表
CREATE TABLE reports (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  title VARCHAR(500) NOT NULL,
  authors JSONB,
  institution VARCHAR(200),
  publish_date DATE,
  abstract TEXT,
  key_points JSONB,
  tags JSONB,
  quality_score INTEGER CHECK (quality_score >= 0 AND quality_score <= 100),
  file_path VARCHAR(500),
  file_size INTEGER,
  page_count INTEGER,
  status VARCHAR(20) DEFAULT 'pending', -- pending, parsing, completed, failed
  created_at TIMESTAMP DEFAULT NOW()
);

-- 热点表
CREATE TABLE hot_topics (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  title VARCHAR(500) NOT NULL,
  source VARCHAR(200),
  source_url TEXT,
  publish_time TIMESTAMP,
  hot_score INTEGER,
  sentiment VARCHAR(20), -- positive, negative, neutral
  related_topics JSONB,
  created_at TIMESTAMP DEFAULT NOW()
);

-- 素材表
CREATE TABLE assets (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  title VARCHAR(500) NOT NULL,
  type VARCHAR(50), -- report, data, quote, news
  content TEXT,
  source VARCHAR(200),
  source_url TEXT,
  tags JSONB,
  usage_count INTEGER DEFAULT 0,
  created_at TIMESTAMP DEFAULT NOW()
);
```
