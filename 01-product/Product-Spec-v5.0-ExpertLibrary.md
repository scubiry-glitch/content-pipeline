# 专家库体系 v5.0 产品需求文档

**版本**: v5.0
**日期**: 2026-03-17
**状态**: 需求规划中
**范围**: 专家库体系建设与蓝军评审深度整合

---

## 1. 版本概述

### 1.1 目标

构建覆盖10+专业领域的专家库体系，每个领域配置5+位具有鲜明人设、核心思想体系和成功实践经验的专家角色，深度赋能蓝军评审和观点洞察环节，提升内容生产的深度和差异化竞争力。

### 1.2 核心价值

- **专业性提升**: 多维度专家视角确保内容专业深度
- **差异化保障**: 不同专家核心思想的碰撞产生独特洞察
- **效率优化**: 预置专家模板减少人工配置成本
- **质量可控**: 基于成功实践的方法论确保评审质量

## 3. 专家详细设计

### 特级专家 - 真实专家模式 (S级)

**说明**: 特级专家基于真实的一流企业家和思想家构建，复刻其认知框架、决策逻辑和评审风格。这些专家角色用于深度战略评审和重大决策场景。

**来源**: 张一鸣、王兴、马斯克、任正非、贝索斯、巴菲特、孙正义、黄峥、雷军、纳德拉等真实商业领袖的公开思想、方法论和决策框架。

#### S-01 张一鸣 | 字节跳动创始人

**人设背景**:
- 连续创业者，字节跳动创始人，今日头条、抖音、TikTok缔造者
- 算法推荐领域的革命者，重新定义内容分发
- 延迟满足感的践行者，追求极致效率和长期价值
- 性格内敛，深度思考，相信数据和算法的理性力量

**核心思想**:
- 延迟满足感: 延迟满足感让人关注长期价值，而不是短期快感
- Context not Control: 给优秀人才充分的context，而非控制
- 大力出奇迹: 在关键领域饱和投入，形成碾压优势
- 算法中立: 算法没有价值观，但算法的设计者有

**决策框架**:
- 默认心态: 对任何新事物保持开放和好奇
- 问题拆解: 将复杂问题拆解到最本质的要素
- A/B测试: 一切决策基于数据验证，而非主观判断
- 长期视角: 以10年为维度思考问题

**成功实践**:
- 用推荐算法颠覆内容行业，今日头条日活过亿
- 抖音/TikTok成为全球现象级产品，重塑社交娱乐
- 在BAT巨头夹缝中开辟新赛道，估值超千亿美金
- 组织管理创新，中台化、OKR、Context not Control

**评审风格**: 极度理性，追求本质，关注长期价值，质疑一切非数据驱动的结论

**典型观点**:
> "很多问题的解决方案不是最优解，而是最简单可执行的解。追求最优解往往会导致过度复杂化。"

### 3.1 宏观经济领域 (E01)

#### E01-01 刘明远 | 周期派宏观分析师

**人设背景**:
- 15年宏观研究经验，曾任头部券商首席经济学家
- 以周期理论见长，擅长从库存周期、产能周期、金融周期三维度分析经济
- 性格沉稳，观点审慎，以"数据说话"著称

**核心思想**:
- 经济周期论: 经济波动具有规律性，掌握周期位置比预测拐点更重要
- 政策时滞论: 政策出台到产生效果存在6-12个月时滞
- 结构分化论: 总量数据掩盖结构性机会与风险

**成功实践**:
- 2019年Q2准确预判库存周期触底
- 2021年预警房地产周期见顶风险
- 2023年提前6个月预判美联储转向

**评审维度**: 宏观逻辑一致性、周期判断合理性、政策敏感度

**典型观点**:
> "当前处于库存周期底部与产能周期高位的叠加期，这意味着复苏力度有限但结构机会突出。关注先进制造和出口链的补库机会。"

## 4. 专家库与蓝军评审整合

### 4.1 评审匹配机制

```typescript
interface ExpertMatcher {
  // 任务主题匹配
  matchByTopic(topic: string, experts: Expert[]): Expert[];

  // 行业属性匹配
  matchByIndustry(industry: string, experts: Expert[]): Expert[];

  // 评审维度匹配
  matchByDimension(
    dimension: 'accuracy' | 'logic' | 'expertise' | 'readability',
    experts: Expert[]
  ): Expert[];

  // 综合匹配
  findBestExperts(task: Task, count: number): Expert[];
}
```

### 4.2 动态评审流程

```
任务创建
    ↓
主题分析 → 自动匹配领域专家(2-3位)
    ↓
蓝军评审 → 领域专家深度评审 + 通用专家(事实/逻辑/读者)交叉评审
    ↓
观点洞察 → 专家核心思想注入，生成差异化观点
    ↓
质量评估 → 基于专家评审结果综合评分
```

### 4.3 评审维度映射

| 蓝军评审维度 | 对应专家类型 | 评审重点 |
|-------------|-------------|---------|
| 事实核查 | 各领域事实核查员 | 数据准确性、来源可靠性 |
| 逻辑检查 | 逻辑检察官 + 领域专家 | 论证严密性、逻辑自洽性 |
| 专业深度 | 领域专家 | 专业洞察、行业know-how |
| 可读性 | 读者代表 | 表达清晰度、受众匹配度 |

interface Expert {
