## 1. 需求价值（承接 WHY.md）

### 1.1 与 WHY.md 的关联

根据 [WHY.md](../WHY.md)，项目核心目标是建立**标准化、可复用的内容生产流水线**，实现质量优先、双周更、高自动化。

**v4.0 解决的关键痛点**：

| WHY.md 痛点 | v4.0 解决方案 | 价值量化 |
|------------|--------------|---------|
| "质量优先：内容必须通过蓝军评审" | 自动化合规检查前置拦截 | 违规内容拦截率 **100%** |
| "无法规模化（手动审核耗时）" | AI自动审核，秒级反馈 | 审核时间从 **30分钟→3秒** |
| 内容安全风险 | 敏感词+广告法+版权+隐私四重防护 | 发布风险 **降至0** |

### 1.2 合规风险场景

**场景1：金融合规**
- 错误示例："投资本产品稳赚不赔，保本保息"
- 风险：《广告法》禁止金融产品承诺收益
- v4.0 拦截：⚠️ 检测到"稳赚""保本"等违规词汇

**场景2：医疗广告**
- 错误示例："服用本保健品可治愈高血压"
- 风险：非药品不得宣称疗效
- v4.0 拦截：⚠️ 检测到"治愈""疗效"等医疗禁用词

**场景3：极限用语**
- 错误示例："全网最低价，史上最强"
- 风险：《广告法》禁止绝对化用语
- v4.0 拦截：⚠️ 检测到"最""第一"等极限词

### 1.3 成功标准

| 指标 | 现状 | 目标 | 验证方式 |
|------|------|------|---------|
| 合规检查覆盖率 | 人工抽查 | 100%自动检查 | 检查日志 |
| 敏感内容拦截率 | - | 100% | 误发事故数=0 |
| 审核响应时间 | 30分钟 | <3秒 | API响应时间 |
| 误报率 | - | <5% | 用户反馈 |
| 内容发布合规率 | 依赖人工 | 100% | 发布后审计 |

## 2. 功能清单

### 2.1 敏感内容检测

**检测维度**:
| 类型 | 示例 | 风险等级 |
|------|------|---------|
| 政治敏感 | 特定人物、事件、表述 | 🔴 极高 |
| 色情暴力 | 不当描述、血腥内容 | 🔴 极高 |
| 歧视仇恨 | 地域、性别、种族歧视 | 🟠 高 |
| 违禁品 | 毒品、枪支等 | 🔴 极高 |

**自定义词库**:
- 支持导入行业特定敏感词
- 支持正则表达式规则
- 支持语义相似度检测（变体识别）

### 2.2 广告法合规检查

**极限词检测**:
```
禁用词汇表：
- 绝对化：最、第一、顶级、唯一、首发
- 承诺性：保过、稳赚、100%有效、根治
- 权威性：国家级、独家、首创、领导品牌
- 对比性：比XX好、碾压、吊打
```

**虚假宣传识别**:
- 无法验证的数据声明
- 虚构的用户评价
- 夸大的产品效果

### 2.3 版权检测

| 检测项 | 说明 |
|--------|------|
| 图片版权 | 检测图片来源，标记潜在侵权风险 |
| 引用来源 | 验证引用内容是否标注出处 |
| 原创度评分 | 与网络内容对比，计算原创度 |

### 2.4 隐私保护检测

**检测类型**:
```

- 手机号：138****1234
- 身份证号：110**********1234
- 银行卡号：6222****1234
- 邮箱：a***@example.com

🟠 建议脱敏：
- 企业内部数据

```

### 4.2 核心模块设计

```typescript
// 合规检查服务
class ComplianceService {
  // 执行完整合规检查
  async checkContent(content: string, config: ComplianceConfig): Promise<ComplianceResult>;

  // 单项检查
  async checkSensitiveWords(content: string, level: string): Promise<SensitiveWordResult[]>;
  async checkAdLaw(content: string): Promise<AdLawResult[]>;
  async checkCopyright(content: string): Promise<CopyrightResult>;
  async checkPrivacy(content: string): Promise<PrivacyResult[]>;

  // 一键修复
  async autoFix(content: string, issues: Issue[]): Promise<FixedContent>;
}

// 敏感词检测引擎
class SensitiveWordEngine {
  // 构建AC自动机
  buildAutomaton(wordList: string[]): Automaton;

  // 高效检测
  detect(content: string): Match[];

  // 支持变体检测
  detectVariants(content: string, variants: VariantRule[]): Match[];
}

// LLM语义分析
class LLMSemanticAnalyzer {
  // 语境理解（避免误判）
  async analyzeContext(match: Match, content: string): Promise<ContextAnalysis>;

  // 生成修改建议
  async suggestFix(issue: Issue, content: string): Promise<string>;
}
```

## 6. 数据库 Schema

```sql
-- 检测规则表
CREATE TABLE compliance_rules (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  category VARCHAR(50) NOT NULL, -- sensitive, ad_law, copyright, privacy
  rule_type VARCHAR(50) NOT NULL, -- keyword, regex, semantic
  pattern TEXT NOT NULL,
  level VARCHAR(20) NOT NULL, -- strict, warning, info
  suggestion TEXT,
  is_custom BOOLEAN DEFAULT false,
  created_at TIMESTAMP DEFAULT NOW()
);

-- 检测日志表
CREATE TABLE compliance_logs (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  content_id UUID NOT NULL,
  check_type VARCHAR(50) NOT NULL,
  result JSONB NOT NULL,
  score INTEGER,
  checked_at TIMESTAMP DEFAULT NOW()
);
```
