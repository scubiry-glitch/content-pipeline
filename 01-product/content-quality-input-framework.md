### 热点发现算法

```typescript
// 热点评分 = 传播速度 × 权威度 × 情感强度 × 时效衰减
interface HotTopic {
  title: string;
  sources: NewsSource[];
  velocity: number;      // 传播速度 (分享/小时)
  authority: number;     // 源权威度 (1-10)
  sentiment: number;     // 情感强度 (-1 to 1)
  freshness: number;     // 时效衰减 (1.0 → 0.0)
  score: number;         // 综合得分
}

// 选题建议生成
function generateTopicSuggestion(hotTopic: HotTopic): TopicSuggestion {
  return {
    angle: selectBestAngle(hotTopic),           // 切入角度
    uniquePerspective: findGapInCoverage(hotTopic), // 空白点
    estimatedImpact: predictEngagement(hotTopic),   // 预期影响力
    relatedResearch: findRelatedReports(hotTopic)   // 关联研报
  };
}
```

## 3. 事实核查机制

### 多层验证

```
第一层：AI 自动核查
├── 数据点验证 (统计数字、日期、人名)
├── 引用溯源 (链接有效性、原文匹配)
├── 逻辑一致性检查
└── 常识冲突检测

第二层：结构化数据交叉验证
├── 财报数据 ↔ 官方披露
├── 市场数据 ↔ 交易所行情
├── 人物信息 ↔ 企业工商信息
└── 历史事件 ↔ 权威记录

第三层：人工确认清单
├── 关键数据来源截图存档
├── 争议性观点多方求证
└── 预测性声明免责声明
```

### 可信度评分

```typescript
interface CredibilityScore {
  // 数据来源可信度
  sourceReliability: number;    // 0-100

  // 可验证性
  verifiability: number;        // 有多少数据可追溯

  // 一致性
  internalConsistency: number;  // 内部逻辑一致性
  externalConsistency: number;  // 与已知事实一致性

  // 透明度
  sourceTransparency: number;   // 来源标注完整度
  methodologyClarity: number;   // 方法论清晰度

  // 总分
  overall: number;
}
```

## 4. 读者匹配度分析

### 受众画像输入

```typescript
interface AudienceProfile {
  // 基础属性
  demographics: {
    ageRange: [number, number];
    education: string[];
    industry: string[];
    role: string[];           // 学生/从业者/管理者
  };

  // 阅读习惯
  readingHabits: {
    preferredLength: 'short' | 'medium' | 'long';
    readingTime: string[];    // 早/中/晚/深夜
    contentFormat: string[];  // 图文/视频/音频
  };

  // 兴趣图谱
  interestGraph: {
    coreTopics: string[];     // 核心关注领域
    relatedTopics: string[];  // 延伸兴趣
    avoidTopics: string[];    // 避开领域
  };

  // 认知水平
  expertiseLevel: 'beginner' | 'intermediate' | 'advanced';
  familiarityWithTopic: number;  // 0-100
}
```

### 内容-受众匹配度

| 匹配维度 | 评估指标 | 优化建议 |
|----------|----------|----------|
| 专业度匹配 | 术语密度 vs 受众认知 | 自动标注生僻术语 |
| 长度匹配 | 阅读时间 vs 习惯时长 | 智能摘要/扩展建议 |
| 深度匹配 | 分析层次 vs 受众水平 | 分层内容结构 |
| 兴趣匹配 | 话题相关性 | 关联推荐 |

## 相关问题

### 需要确认

1. **新闻源范围**: 聚焦财经科技？还是扩展到更多领域？
2. **更新频率**: 实时监控还是定时汇总？
3. **竞品范围**: 监测哪些账号/媒体作为对标？
4. **受众定义**: 先服务哪类读者（小白/进阶/专业）？

### 下一步建议

1. **Day 1-2**: 搭建 RSS 聚合器，接入 5-10 个核心源
2. **Day 3**: 实现热点发现算法，输出选题建议
3. **Day 4-5**: 接入事实核查，标注数据可信度
4. **Week 2**: 竞品监控 + 差异化分析
