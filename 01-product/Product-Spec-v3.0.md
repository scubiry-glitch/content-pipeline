# 产品规格: 内容质量输入体系 v3.0

## 概述

**背景**: v2.0 实现了内容生成流水线，但输出质量受限于输入质量。garbage in, garbage out。

**目标**: 构建多维度内容输入体系，确保每条内容都有高质量的信息源、差异化视角、可信度验证。

**成功标准**:
- 热点发现准确率 > 80%（命中后续爆款话题）
- 事实核查覆盖 100% 数据点
- 内容差异化指数 > 70（与竞品差异度）

---

## 功能规格

### 模块 1: 新闻聚合器 (NewsAggregator)

| 功能 | 优先级 | 验收标准 |
|------|--------|----------|
| RSS 源管理 | P0 | 支持增删改查 RSS 源，自动去重 |
| 定时抓取 | P0 | 每15分钟轮询一次，增量更新 |
| 热点发现 | P0 | 基于传播速度+情感强度计算热点分 |
| 选题建议 | P1 | 根据热点生成立项建议 |

**数据源分级**:
- P0: 官方公告（证监会、央行）- 实时
- P1: 头部财经（财新、第一财经）- 15分钟
- P2: 垂直媒体（36氪、机器之心）- 30分钟

### 模块 2: 事实核查器 (FactChecker)

| 功能 | 优先级 | 验收标准 |
|------|--------|----------|
| 数据点提取 | P0 | 自动提取数字、日期、人名等可验证信息 |
| 多源交叉验证 | P0 | 对关键数据点找到至少2个独立来源 |
| 可信度评分 | P1 | 输出 0-100 可信度分数 |
| 风险提示 | P1 | 低可信度内容标注警告 |

### 模块 3: 差异化分析器 (DifferentiationAnalyzer)

| 功能 | 优先级 | 验收标准 |
|------|--------|----------|
| 竞品监控 | P1 | 抓取对标账号最新内容 |
| 主题聚类 | P1 | 识别已有内容覆盖的角度 |
| 空白发现 | P1 | 找出未被覆盖的切入点 |
| 原创度评分 | P2 | 计算与现有内容的相似度 |

### 模块 4: 读者匹配器 (AudienceMatcher)

| 功能 | 优先级 | 验收标准 |
|------|--------|----------|
| 阅读难度分析 | P1 | 评估文章专业度等级 |
| 平台适配建议 | P2 | 针对不同平台生成优化建议 |
| 阅读时间估算 | P2 | 准确预估阅读时长 |
