# PRD-002: 沉浸式写作模式 (Immersive Writing Mode)

## 1. 背景与目标

### 1.1 问题陈述
当前写作界面存在大量干扰：
- 顶部导航栏、左侧边栏、右侧属性面板分散注意力
- 通知气泡、状态更新打断心流
- 工具按钮太多，写一个字要犹豫用哪个格式
- 看不到同事在做什么，缺乏协作氛围

### 1.2 目标
打造"心流写作"体验：
- **减少视觉噪音**：写作时只剩下文字
- **减少认知负担**：格式工具在需要时出现，不需要时隐藏
- **增强专注感**：时间流逝可视化，产出实时反馈
- **适度协作感**：知道有人在附近，但不打扰

### 1.3 成功指标
- 用户单次连续写作时长从 15 分钟 → 45 分钟
- 写作模式下的任务完成率提升 40%
- 用户主动进入沉浸模式的占比 > 50%

## 2. 用户故事

**场景 A：开始深度写作**
```
作为：专栏作者
当我：需要写一篇深度长文
我希望：一键进入一个只有我和文字的世界
而不是：在各种面板和通知中挣扎
```

**场景 B：引用素材**
```
作为：内容创作者
当我：写作时需要引用研报数据
我希望：不用离开当前页面就能拖拽插入
而不是：打开素材库 → 找到文件 → 复制 → 回来粘贴
```

**场景 C：与 AI 协作**
```
作为：内容创作者
当我：写到一半卡壳了
我希望：AI 像同事一样在旁边，需要时才说话
而不是：我主动打开一个对话框去问
```

**场景 D：写作马拉松**
```
作为：专栏作者
当我：开始一场长时间写作
我希望：看到时间流逝和产出进度
以便：保持动力和节奏感
```

## 3. 功能设计

### 3.1 三种沉浸级别

```
级别 1: 专注模式 (Focus Mode)
        保留极简工具栏，仅隐藏导航
        
级别 2: 禅模式 (Zen Mode)  
        全屏，只剩下光标和文字
        
级别 3: 深度流动 (Deep Flow)
        全屏 + 背景音乐 + 番茄钟 + 产出追踪
```

### 3.2 专注模式 (Focus Mode)

#### 触发方式
- 快捷键：`⌘ + Enter`
- 按钮：编辑器右上角「专注」图标
- 自动：连续写作 5 分钟后询问是否进入

#### 界面变化
```
Before:
┌──────────────────────────────────────────────────────────────┐
│ [Logo] 导航栏 [搜索] [通知] [设置]                            │
├──────────┬───────────────────────────────┬─────────────────┤
│ 素材列表 │                               │ 属性面板        │
│          │      编辑器（中间）            │ • 标签          │
│          │                               │ • 状态          │
│          │                               │ • 字数统计      │
└──────────┴───────────────────────────────┴─────────────────┘

After (Focus Mode):
┌──────────────────────────────────────────────────────────────┐
│              极简顶部（可隐藏）                               │
│              [退出] [字数] [保存状态]                         │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│                                                              │
│                    编辑器（占满屏幕）                         │
│                    内容居中， comfortable 宽度                │
│                                                              │
│                                                              │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

#### 浮动工具栏
- 默认隐藏，鼠标靠近顶部时滑出
- 包含：保存状态、字数统计、退出按钮
- 选中文字时，出现格式工具条

### 3.3 禅模式 (Zen Mode)

#### 触发方式
- 快捷键：`⌘ + Shift + Enter`
- 从专注模式再次点击「禅模式」按钮

#### 界面特点
```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│                                                             │
│                                                             │
│                                                             │
│                只有文字，连滚动条都隐藏                      │
│                                                             │
│                光标在屏幕中央固定位置                        │
│                文字像打字机一样从下往上推                    │
│                                                             │
│                                                             │
│                                                             │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

#### 禅模式特有功能
- **打字机滚动**：光标始终保持在屏幕垂直中央
- **打字音效**：机械键盘声音（可关闭）
- **呼吸灯**：边缘微弱脉动，帮助保持节奏
- **仅显示当前段落**：其他段落淡出（可选）

### 3.4 深度流动模式 (Deep Flow)

在禅模式基础上增加：

#### 时间可视化
```
屏幕底部微弱进度条：
[████████████████░░░░░░░░░░]  45分钟 / 60分钟目标

或圆形时钟：
      ╭──────────╮
     ╱   45:23    ╲
    │  ██░░░░░░░  │  ← 完成度
    │   writing   │
     ╲           ╱
      ╰──────────╯
```

#### 产出仪表盘（透明悬浮）
```
┌────────────────────┐
│  本次写作           │
│  时间: 45 分钟      │
│  字数: 1,234        │
│  速度: 27 字/分钟   │
│  质量分: 82         │
│                    │
│  [AI 续写建议 ✨]   │
└────────────────────┘

显示位置：右下角，低透明度，鼠标移入高亮
```

#### 协作感知（Ghost Presence）
```
屏幕边缘微弱光点：
● 同事 A 正在看这篇文档（蓝色微光）
● 同事 B 也在写作模式（绿色微光）

不显示具体位置、不弹通知，只是"有人陪伴"的感觉
```

### 3.5 素材悬浮面板 (Floating Asset Panel)

#### 设计目标
写作时无需离开页面即可引用素材

#### 交互
```
快捷键：⌘ + Shift + R

屏幕右侧滑出素材面板：
┌──────────────────────────┐
│ 🔍 搜索素材...            │
├──────────────────────────┤
│ 📌 最近使用               │
│ • 2024年新能源报告        │
│ • 央行降准政策原文        │
│ • 张一鸣演讲全文          │
├──────────────────────────┤
│ 📂 当前任务相关           │
│ • 素材1                   │
│ • 素材2                   │
├──────────────────────────┤
│ 🤖 AI 推荐                │
│ 根据当前内容推荐相关素材   │
└──────────────────────────┘

拖拽素材到编辑器 = 引用插入
双击素材 = 预览浮窗
```

#### 智能引用
```typescript
// 当用户输入特定内容时，自动推荐素材
const autoSuggest = {
  trigger: "2024年新能源汽车",
  suggestion: "检测到您提到新能源汽车，推荐引用：《2024年新能源汽车市场报告》"
}
```

### 3.6 AI 写作助手 (Ambient AI)

#### 设计原则
- AI 像安静的同事坐在旁边
- 不主动打扰，但在需要时随时可用
- 通过微妙视觉提示表示"我在这里"

#### 视觉表现
```
屏幕右下角微弱发光点：

● 绿灯：AI 已准备好，可以询问
● 呼吸灯：AI 正在分析您的写作
● 闪烁：AI 有建议（检测到卡壳/可优化点）

点击或 ⌘ + / 展开对话：
┌─────────────────────┐
│ AI 助手              │
│ ┌─────────────────┐ │
│ │ 需要我帮您续写  │ │
│ │ 这一段吗？      │ │
│ └─────────────────┘ │
│ [续写] [忽略]       │
│                     │
│ [输入指令...]       │
└─────────────────────┘
```

#### 主动干预时机
| 场景 | AI 行为 | 用户感知 |
|------|---------|---------|
| 5分钟无输入 | 微弱闪烁 | "需要灵感吗？" |
| 检测到写作困难 | 弹出建议卡片 | "检测到卡壳，试试..." |
| 引用过期数据 | 高亮 + 提示 | "这个数据已更新，最新是..." |
| 质量分下降 | 侧边提示 | "这段可读性较低，建议..." |

## 4. 技术实现

### 4.1 状态管理
```typescript
interface ImmersiveState {
  mode: 'normal' | 'focus' | 'zen' | 'deep_flow';
  
  // 专注模式配置
  config: {
    showWordCount: boolean;
    showProgress: boolean;
    typewriterScroll: boolean;
    typingSound: boolean;
    backgroundMusic: boolean;
    showGhostPresence: boolean;
  };
  
  // 本次写作会话
  session: {
    startTime: string;
    wordCount: number;
    targetDuration: number; // 番茄钟目标
    breaks: number;
  };
}
```

### 4.2 键盘快捷键
```typescript
const shortcuts = {
  '⌘ + Enter': '进入/退出 专注模式',
  '⌘ + Shift + Enter': '进入/退出 禅模式',
  '⌘ + Shift + R': '切换素材悬浮面板',
  '⌘ + /': '聚焦 AI 助手',
  'Esc': '退出当前沉浸级别（逐级退出）',
  '⌘ + S': '保存（有自动保存，但给用户安全感）',
  '⌘ + Shift + F': '全屏/退出全屏',
}
```

### 4.3 性能优化
- 进入沉浸模式时，暂停非必要的数据同步
- 减少 React 重渲染，使用 shouldComponentUpdate
- 打字机滚动使用 transform 而非 scrollTop
- 背景音效使用 Web Audio API，低延迟
