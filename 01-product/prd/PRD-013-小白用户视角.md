# PRD-013: 小白用户零门槛体验 (Zero Barrier for Beginners)

> "如果我妈不会用，这个产品就是失败的。"
> —— 产品经理的朴素真理

## 1. 小白用户画像

### 1.1 典型用户：小李

```
背景：
├─ 某传统行业市场部员工
├─ 公司要求开始做内容营销
├─ 没有写作经验
├─ 对 AI 工具完全陌生
├─ 害怕技术，怕点错东西
└─ 目标：每周产出 2-3 篇专业内容

痛点：
├─ 不知道写什么（没有选题灵感）
├─ 不会写专业内容（不懂行话）
├─ 不知道怎么用工具（看到复杂界面就慌）
├─ 不敢发布（怕写错被领导批评）
├─ 不知道效果好不好（没有数据概念）
└─ 遇到问题没人问（客服响应慢）

期望：
├─ 像有个专家在旁边手把手教
├─ 每一步都有明确指引
├─ 错了能撤销，不会搞砸
├─ 能快速看到成果（正反馈）
└─ 价格不贵（个人付费或公司小额预算）
```

### 1.2 使用场景

```
场景：小李第一次使用

第 1 分钟：打开系统
├─ 期望：知道该点哪里
└─ 恐惧：点错会不会删数据

第 5 分钟：创建第一个任务
├─ 期望：有明确步骤，不会迷路
└─ 恐惧：选错类型会不会浪费 AI 次数

第 15 分钟：看到 AI 生成内容
├─ 期望：哇，我真的写出来了
└─ 恐惧：这能用吗？会不会太假？

第 30 分钟：发布第一篇内容
├─ 期望：有成就感，想发朋友圈
└─ 恐惧：发布后还能改吗？

第 1 天：收到数据反馈
├─ 期望：知道大家喜不喜欢
└─ 恐惧：数据不好是不是我能力问题
```

## 2. 零门槛设计原则

### 2.1 三次点击原则

```
任何核心功能，小白用户在 3 次点击内完成。

例子：写一篇文章
├─ 点击 1："开始写作"（首页最显眼位置）
├─ 点击 2：输入想法/选择推荐选题
└─ 点击 3："生成"按钮

然后等结果。就这么简单。
```

### 2.2 零学习成本

```
不需要：
❌ 看教程视频
❌ 读帮助文档
❌ 理解专业术语
❌ 学习快捷键
❌ 配置复杂参数

做到：
✅ 打开即用
✅ 像聊天一样自然
✅ 错误可撤销
✅ 每一步都有引导
✅ 术语有解释
```

### 2.3 安全感设计

```
小白最怕"搞砸了"。

安全设计：
├─ 所有操作可撤销
├─ 重要操作二次确认（但不过度）
├─ 自动保存，不怕断网
├─ 预览模式，发布前可见效果
├─ 温和的错误提示，不说"错误"说"需要调整"
└─ 明确的"返回上一步"按钮
```

## 4. 界面简化

### 4.1 小白模式 vs 专家模式

```
默认进入"小白模式"，随时可以切换到"专家模式"。

小白模式特点：
├─ 只显示核心功能（隐藏高级选项）
├─ 每一步有引导（不迷路）
├─ 术语解释（鼠标悬停显示说明）
├─ 智能默认值（不需要配置）
└─ 一键操作（减少选择）

专家模式：
└─ 显示全部功能，自由配置
```

### 4.2 术语解释系统

```
所有专业术语都有解释：

鼠标悬停"质量分"：
┌──────────────────────────────────┐
│  💡 质量分                        │
│                                   │
│  AI 从 5 个维度评估文章质量：     │
│  • 数据准确性                     │
│  • 逻辑清晰度                     │
│  • 可读性                         │
│  • 原创性                         │
│  • 热点相关性                     │
│                                   │
│  80分以上 = 质量不错，可以发布    │
│  90分以上 = 优秀，可能成爆款      │
└──────────────────────────────────┘
```

### 4.3 防错设计

```
可能出错的地方：

1. 误删除
   ├─ 删除前确认："真的要删除吗？删除后可以在回收站找回"
   └─ 回收站保留 30 天

2. 发布错误内容
   ├─ 发布前预览："这是最终效果，确认发布吗？"
   └─ 发布后 5 分钟内可撤回

3. 选错参数
   ├─ 智能检测："检测到您选择了'学术论文'类型，但内容是科普风格，建议改为'科普文章'"
   └─ 一键修正

4. 网络中断
   ├─ 自动保存：每 30 秒自动保存
   └─ 恢复提示："检测到未完成的编辑，是否继续？"
```

## 5. 帮助系统

### 5.1 随时可问

```
右下角固定"问 AI 助手"按钮：

用户："这个按钮是干嘛的？"
AI："这是'热点匹配'功能，它会分析你写的内容和当前热点的相关性。
     匹配度越高，文章被看到的机会越大。
     你的文章匹配度 85%，说明踩中热点了！"

用户："我能再改改标题吗？"
AI："当然可以！点击标题就能编辑。
     或者我可以帮你生成几个备选标题，你选一个？"

用户："我不知道写什么了..."
AI："没关系，看看这些方向：
     [推荐 3 个选题]
     或者告诉我你感兴趣的话题，我帮你找角度。"
```

### 5.2 视频教程（可选）

```
不是强制看，是"需要时出现"。

场景 1：用户停留在一个页面超过 30 秒
提示："需要我演示这个功能怎么用吗？" [看视频] [不用]

场景 2：用户第一次使用某功能
提示："这是第一次使用 XX 功能，需要快速上手教程吗？"

视频特点：
├─ 不超过 2 分钟
├─ 没有废话，直接演示
├─ 可以 1.5 倍速
├─ 随时可关闭
└─ 文字版对照（不方便看视频的可以看文字）
```
