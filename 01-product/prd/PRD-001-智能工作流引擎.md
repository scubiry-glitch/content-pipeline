# PRD-001: 智能工作流引擎 (Smart Workflow Engine)

## 1. 背景与目标

### 1.1 问题陈述
当前系统是一个"被动工具"，用户需要主动寻找该做什么：
- 打开仪表盘 → 看统计数据 → 点击进入任务列表 → 找到具体任务 → 开始工作
- 用户经常在多个模块间迷失，不清楚当前优先级最高的事项
- 缺乏基于时间、状态、内容的智能推荐

### 1.2 目标
将系统从"工具思维"转变为"助理思维"：
- **减少决策疲劳**：系统告诉用户"现在该做什么"
- **缩短操作路径**：一键直达，无需导航
- **提升产出效率**：基于数据智能推荐最佳行动

### 1.3 成功指标
- 用户找到"下一步行动"的平均点击次数：从 4 次 → 1 次
- 日活跃用户完成任务的转化率提升 30%
- 用户平均停留时间减少 20%（效率提升，而非流失）

## 3. 功能设计

### 3.1 核心概念：Next Action Card（下一步行动卡片）

在仪表盘顶部固定展示一张「智能行动卡片」，根据算法动态变化。

#### 卡片结构
```typescript
interface NextActionCard {
  id: string;
  priority: 'urgent' | 'high' | 'normal';
  type: 'review' | 'write' | 'research' | 'publish' | 'switch' | 'break';
  
  // 主标题
  title: string; // "审阅 AI 生成的初稿"
  
  // 副标题/上下文
  subtitle: string; // "《新能源出海战略分析》- 昨晚 23:00 生成"
  
  // 行动按钮（主 + 次）
  primaryAction: {
    label: string; // "立即审阅"
    target: string; // 路由或回调
  };
  secondaryAction?: {
    label: string; // "15分钟后再说"
    action: 'snooze' | 'dismiss';
  };
  
  // 支撑数据
  metadata: {
    estimatedTime: number; // 预计耗时 15 分钟
    aiConfidence: number; // AI 推荐置信度 92%
    contextInfo: string[]; // ["距发布DDL还有3小时", "热点相似度85%"]
  };
  
  // 视觉标识
  visualCue: {
    icon: string;
    color: string;
    animation?: string;
  };
}
```

### 3.2 推荐算法规则

#### 时间触发规则
| 时间段 | 推荐类型 | 示例 |
|--------|---------|------|
| 09:00-10:00 | 昨日回顾 | "您有 3 个昨晚 AI 生成的初稿待审阅" |
| 11:00-12:00 | 高效产出 | "上午是您的黄金时段，建议完成复杂写作任务" |
| 14:00-15:00 | 低能耗任务 | "下午效率较低，建议处理素材整理类任务" |
| 17:00-18:00 | 收尾提醒 | "还有 1 个任务待发布，预计耗时 10 分钟" |
| 周五 16:00+ | 周报生成 | "本周已完成 5 篇，生成周报？" |

#### 状态触发规则
| 状态 | 触发条件 | 推荐动作 |
|------|---------|---------|
| 逾期 | 任务超过 Due Date | 高优提醒 + 延期/加速选项 |
| 停滞 | 2 天无进度更新 | 建议分配给专家 + 推荐专家列表 |
| 热点匹配 | 任务主题与热点相似度 > 80% | 建议加速发布 + 热点数据预览 |
| AI 完成 | AI 生成内容就绪 | 审阅提醒 + 预计审阅时间 |
| 素材就绪 | 引用的外部素材已更新 | 更新提醒 + 差异对比 |

### 3.3 交互设计

#### 主界面布局
```
┌─────────────────────────────────────────────────────────────┐
│  智能行动卡片（Next Action Card）← 固定在顶部，永远可见       │
│  ┌──────────────────────────────────────────────────────┐   │
│  │  🔥 紧急  │  《央行降准分析》初稿待审阅（AI生成）      │   │
│  │           │  预计耗时：15分钟  │  距截止：3小时       │   │
│  │           │  [立即审阅]  [延后15分钟]  [分配给专家]    │   │
│  └──────────────────────────────────────────────────────┘   │
├─────────────────────────────────────────────────────────────┤
│  其他仪表盘内容...                                            │
└─────────────────────────────────────────────────────────────┘
```

#### 卡片状态流转
```
[用户打开页面]
    ↓
[算法计算最优行动] → 如果无明确推荐 → 显示"自由探索模式"
    ↓
[展示 Next Action Card]
    ↓
用户操作：
├── 点击主按钮 → 直接进入工作流
├── 点击"延后" → 卡片消失，30分钟后再次出现
├── 点击"忽略" → 记录原因，优化未来推荐
└── 完成动作 → 自动展示下一个推荐
```

### 3.4 负反馈处理

当用户忽略推荐时，收集原因：
- "这不是我想做的" → 调整算法权重
- "我现在没时间" → 延后提醒
- "这个任务不急" → 降低优先级标记
- "已完成" → 更新任务状态

### 4.2 算法伪代码
```typescript
function calculateNextAction(userId: string): NextActionCard | null {
  const context = {
    time: new Date(),
    pendingTasks: getPendingTasks(userId),
    userPref: getUserPreference(userId),
    hotTopics: getActiveHotTopics(),
    aiGeneratedContent: getReadyAIContent(userId),
  };
  
  const candidates = [];
  
  // 1. 逾期任务（最高优先级）
  const overdue = context.pendingTasks.filter(t => isOverdue(t));
  if (overdue.length > 0) {
    candidates.push(createOverdueCard(overdue[0]));
  }
  
  // 2. 热点匹配任务
  const hotTopicMatch = context.pendingTasks
    .filter(t => matchHotTopic(t, context.hotTopics) > 0.8)
    .sort(byMatchScore)[0];
  if (hotTopicMatch) {
    candidates.push(createHotTopicCard(hotTopicMatch));
  }
  
  // 3. AI 生成内容待审阅
  const readyAI = context.aiGeneratedContent.filter(c => c.status === 'pending_review')[0];
  if (readyAI) {
    candidates.push(createReviewCard(readyAI));
  }
  
  // 4. 基于用户习惯的推荐
  const habitBased = recommendByHabit(context);
  if (habitBased) {
    candidates.push(habitBased);
  }
  
  // 按优先级排序，返回第一个
  return candidates.sort(byPriority)[0] || null;
}
```

## 5. 验收标准

### 5.1 功能验收
- [ ] 用户打开仪表盘，始终看到 Next Action Card
- [ ] 卡片内容根据时间、状态、内容动态变化
- [ ] 点击主按钮一键进入对应工作流
- [ ] 支持"延后""忽略"等负反馈操作
- [ ] 收集反馈数据用于算法优化

### 5.2 性能验收
- [ ] 推荐计算 < 100ms
- [ ] 卡片渲染 < 50ms
- [ ] 不阻塞页面其他内容加载

### 5.3 体验验收
- [ ] 推荐的接受率 > 60%（A/B Test 对比无推荐组）
