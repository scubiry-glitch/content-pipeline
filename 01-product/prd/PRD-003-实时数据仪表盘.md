# PRD-003: 实时数据仪表盘 (Real-time Dashboard)

## 1. 背景与目标

### 1.1 问题陈述
当前仪表盘数据滞后、缺乏反馈：
- 不知道今天的产出如何
- 不知道和昨天比有没有进步
- 不知道在团队里是什么水平
- 看不到实时数据，缺乏成就感

### 1.2 目标
打造"游戏化"的实时反馈系统：
- **实时可见**：产出数据实时更新
- **对比反馈**：与昨日、上周、团队对比
- **成就感知**：完成有庆祝，连续有奖励
- **预测指导**：AI 预测今日完成量

### 1.3 成功指标
- 用户每日查看仪表盘次数 > 3 次
- 用户主动分享"今日成就"占比 > 20%
- 数据更新延迟 < 1 秒

## 4. 技术实现

### 4.1 数据架构
```typescript
// 用户实时数据
interface UserRealtimeStats {
  userId: string;
  date: string;
  
  // 今日数据
  today: {
    wordCount: number;
    writingDuration: number; // 分钟
    qualityScore: number;
    tasksCompleted: number;
    tasksTarget: number;
  };
  
  // 对比数据
  comparison: {
    vsYesterday: {
      wordCountDelta: number;  // +15%
      qualityDelta: number;    // -3
    };
    vsLastWeek: {
      avgWordCountDelta: number;
    };
  };
  
  // AI 预测
  prediction: {
    estimatedTotal: number;
    confidence: number;
    recommendation: string;
  };
  
  // 团队排名
  teamRank: {
    current: number;
    total: number;
    trend: 'up' | 'down' | 'stable';
  };
  
  // 更新时间
  lastUpdated: string;
}
```

### 4.2 实时推送
```typescript
// WebSocket 连接
const ws = new WebSocket('wss://api.content-pipeline.com/realtime');

// 订阅个人数据
ws.send(JSON.stringify({
  action: 'subscribe',
  channels: ['user:stats', 'team:leaderboard', 'achievements']
}));

// 接收推送
ws.onmessage = (event) => {
  const update = JSON.parse(event.data);
  updateHUD(update);  // 更新右上角 HUD
  updateDashboard(update);  // 更新仪表盘
  
  // 成就解锁
  if (update.type === 'achievement_unlocked') {
    showAchievementModal(update.achievement);
  }
};
```

### 4.3 性能优化
- **增量更新**：只推送变化的数据，不推全量
- **本地缓存**：最近 5 分钟数据缓存，离线时显示
- **批量渲染**：100ms 内的多个更新合并渲染
- **降级策略**：WebSocket 失败时轮询（5 秒一次）
