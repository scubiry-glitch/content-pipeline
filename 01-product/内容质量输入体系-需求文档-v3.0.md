# 内容质量输入体系 - 产品需求文档 v3.0

**版本**: v3.0
**模块**: 内容质量输入 (v3.4)
**日期**: 2026-03-17
**状态**: 开发中

---

## 1. 需求概述

### 1.1 目标
构建完整的内容质量输入体系，实现研报上传、解析、搜索、热点追踪、智能匹配、素材管理等核心功能，为内容生产流水线提供高质量的内容输入。

### 1.2 价值
- 提升研报利用效率，减少重复劳动
- 实时追踪热点，把握市场动态
- 智能内容匹配，提高创作效率
- 统一管理素材，沉淀内容资产

### 1.3 范围
| 模块 | 功能 | 优先级 |
|------|------|--------|
| 研报上传 | PDF/Word上传、批量上传、上传进度 | P0 |
| 研报解析 | 文本提取、元数据提取、质量评分 | P0 |
| 研报搜索 | 关键词搜索、筛选、分页、排序 | P0 |
| 热点追踪 | 热点列表、趋势图表、RSS管理 | P1 |
| 智能匹配 | 内容相似度、相关推荐 | P1 |
| 素材管理 | 素材提取、引用管理、格式生成 | P1 |

## 2. 功能模块详细需求

### 2.1 研报上传模块

#### 2.1.1 功能描述
支持用户上传PDF和Word格式的研报文件，系统会自动进行解析和质量评分。

#### 2.1.2 用户界面
- **上传入口**: 研报库页面的"上传研报"按钮
- **上传弹窗**: 支持拖拽上传、点击选择文件
- **进度展示**: 上传进度条、解析进度提示
- **结果反馈**: 上传成功/失败提示

#### 2.1.3 功能需求
| ID | 需求描述 | 验收标准 |
|----|----------|----------|
| UP-001 | 支持PDF文件上传 | 可上传.pdf文件，最大50MB |
| UP-002 | 支持Word文件上传 | 可上传.docx文件，最大50MB |
| UP-003 | 支持批量上传 | 一次可选择多个文件上传 |
| UP-004 | 上传后状态为pending | 上传成功后显示"待解析"状态 |
| UP-005 | 文件大小限制 | 超过50MB的文件被拒绝，提示用户 |
| UP-006 | 文件格式校验 | 只接受.pdf和.docx格式 |
| UP-007 | 上传进度显示 | 显示上传进度百分比 |
| UP-008 | 生成唯一研报ID | 每个研报有唯一标识符 |

#### 2.1.4 前端组件
- UploadReportModal: 上传研报弹窗组件
- UploadProgress: 上传进度组件
- FileDropZone: 文件拖拽上传区域

---

### 2.2 研报解析模块

#### 2.2.1 功能描述
自动解析上传的研报文件，提取标题、作者、机构、核心观点、标签等信息，并进行质量评分。

#### 2.2.2 用户界面
- **解析状态**: 在研报列表中显示解析状态
- **解析结果**: 研报详情页展示解析结果
- **质量评分**: 可视化展示质量维度得分

#### 2.2.3 功能需求
| ID | 需求描述 | 验收标准 |
|----|----------|----------|
| PA-001 | 解析PDF文本内容 | 正确提取PDF中的文字 |
| PA-002 | 提取标题 | 准确率>95% |
| PA-003 | 提取作者 | 准确率>90% |
| PA-004 | 提取机构 | 准确率>90% |
| PA-005 | 提取发布日期 | 准确率>85% |
| PA-006 | 生成核心观点 | 提取3-5个核心观点 |
| PA-007 | 生成标签 | 自动打5-10个标签 |
| PA-008 | 计算质量分 | 6个维度评分 |
| PA-009 | 解析状态管理 | pending→parsing→parsed/error |
| PA-010 | 解析时间 | 单份研报<5秒 |

#### 2.2.4 质量维度
| 维度 | 权重 | 说明 |
|------|------|------|
| 权威性 | 25% | 发布机构影响力 |
| 完整性 | 20% | 内容结构完整度 |
| 逻辑性 | 20% | 论证逻辑清晰度 |
| 时效性 | 15% | 发布时间新鲜度 |
| 引用数 | 10% | 被引用次数 |
| 原创性 | 10% | 内容独创性 |

#### 2.2.5 前端组件
- ReportQualityCard: 研报质量评分卡片
- ParseResultPanel: 解析结果展示面板
- QualityRadarChart: 质量维度雷达图

#### 2.3.3 功能需求

| SE-003 | 按质量分筛选 | 可设置最低质量分阈值 |

#### 2.3.4 前端组件

#### 2.5.3 功能需求
| ID | 需求描述 | 验收标准 |
|----|----------|----------|
| MA-001 | 计算内容相似度 | 相似度0-100分 |
| MA-002 | 强相关标记 | >85分标记为强相关 |
| MA-003 | 推荐相关研报 | 推荐5-10篇相关研报 |
| MA-004 | 推荐相关素材 | 推荐相关图表、数据 |
| MA-005 | 关键词匹配 | 提取并匹配关键词 |
| MA-006 | 时间衰减 | 旧内容权重降低 |
| MA-007 | 推荐数量限制 | 最多10条推荐 |
| MA-008 | 响应时间 | <200ms |

#### 2.5.4 前端组件

- MatchScoreBadge: 匹配度徽章
- QuickQuoteButton: 一键引用按钮

## 4. API接口需求

### 4.1 研报接口
```typescript
// 研报上传
POST   /api/v1/reports/upload

// 研报列表
GET    /api/v1/reports?search=&institution=&minQuality=&page=&limit=

// 研报详情
GET    /api/v1/reports/:id

// 触发解析
POST   /api/v1/reports/:id/parse

// 获取解析结果
GET    /api/v1/reports/:id/parse-result

// 获取相关研报
GET    /api/v1/reports/:id/related

// 删除研报
DELETE /api/v1/reports/:id
```

### 4.2 热点接口
```typescript
// 热点列表
GET    /api/v1/quality/hot-topics?trend=&limit=

// 热点详情
GET    /api/v1/quality/hot-topics/:id

// 关注热点
POST   /api/v1/quality/hot-topics/:id/follow

// 取消关注
POST   /api/v1/quality/hot-topics/:id/unfollow

// 获取趋势数据
GET    /api/v1/quality/hot-topics/:id/trend?days=

// RSS源列表
GET    /api/v1/quality/rss-sources

// 添加RSS源
POST   /api/v1/quality/rss-sources

// 触发RSS抓取
POST   /api/v1/quality/rss-sources/crawl
```

### 4.3 素材接口
```typescript
// 素材列表
GET    /api/v1/assets?type=&sourceId=&limit=

// 创建素材
POST   /api/v1/assets

// 从研报提取素材
POST   /api/v1/reports/:id/extract-assets

// 引用素材
POST   /api/v1/assets/:id/quote

// 删除素材
DELETE /api/v1/assets/:id
```
