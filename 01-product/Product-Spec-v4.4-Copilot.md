## 1. 需求价值（承接 WHY.md）

### 1.1 与 WHY.md 的关联

根据 [WHY.md](../WHY.md)，项目核心目标是让内容创作者**专注于"思考"而非"操作工具"**。

**v4.4 解决的关键痛点**：

| WHY.md 痛点 | v4.4 解决方案 | 价值量化 |
|------------|--------------|---------|
| "缺乏有效的生产工具链" | 自然语言配置，零学习成本 | 上手时间从 **数小时→5分钟** |
| "产出效率极低" | AI主动建议，减少决策负担 | 配置时间减少 **80%** |
| "手动操作多个工具" | 一站式智能助手 | 操作步骤减少 **70%** |

### 1.2 Copilot 场景

**场景1：自然语言配置**
```

```

```
✓ 多平台适配: 即刻（短摘要版本）+ 公众号（长文）"
```

### 2.2 主动建议

**干预点配置**:
```typescript
interventionPoints: [
  'stage_start',      // Stage 启动时
  'data_insufficient', // 数据不足时
  'logic_weak',       // 逻辑薄弱时
  'post_generate',    // 内容生成后
  'pre_publish',      // 发布前
]
```

**建议类型**:
| 触发点 | 建议内容 | 示例 |
|--------|---------|------|
| stage_start | 推荐配置 | "基于您近期偏好，建议使用"简洁专业"风格" |
| data_insufficient | 数据补充 | "缺少2024年最新数据，建议补充" |
| logic_weak | 论证增强 | "此处因果关系不够明确，建议添加对比数据" |
| post_generate | 优化建议 | "标题吸引力可提升，建议增加数字" |
| pre_publish | 最终检查 | "检测到未标注引用来源，建议补充" |

### 2.3 智能问答

**问答类型**:
```
Why 类:
- "为什么推荐这个选题？"
- "为什么用这个结构？"
- "为什么选这个时间发布？"

What 类:
- "这个数据的来源是什么？"
- "竞品是怎么写的？"
- "热点趋势如何？"

How 类:
- "怎么提升阅读量？"
- "怎么优化这个标题？"
- "怎么让论证更有力？"
```

### 2.4 一键优化

**优化指令**:
```
标题优化:
- "优化标题吸引力" → 生成5个备选标题
- "标题加点悬念" → 增加悬念元素
- "标题突出数据" → 添加关键数字

内容优化:
- "增强开篇钩子" → 重写开头段落
- "增加案例" → 自动搜索并插入案例
- "优化结尾" → 强化结论和行动号召

格式优化:
- "压缩到3000字" → 智能删减冗余内容
- "增加小标题" → 自动添加结构标签
- "调整段落长度" → 优化阅读节奏
```

### 4.2 核心模块设计

```typescript
// Copilot 服务
class CopilotService {
  // 意图识别与配置生成
  async parseIntent(naturalLanguage: string): Promise<ConfigIntent>;

  // 生成配置
  async generateConfig(intent: ConfigIntent): Promise<PipelineConfig>;

  // 主动建议
  async generateSuggestion(
    context: PipelineContext,
    triggerPoint: string
  ): Promise<Suggestion>;

  // 智能问答
  async answerQuestion(
    question: string,
    context: QaContext
  ): Promise<Answer>;

  // 一键优化
  async optimize(content: string, optimizationType: string): Promise<OptimizedContent>;
}

// 意图识别
class IntentRecognizer {
  // 分类意图
  async classifyIntent(text: string): Promise<IntentType>;

  // 提取实体
  async extractEntities(text: string): Promise<Entities>;

  // 映射参数
  mapToConfig(entities: Entities): Partial<PipelineConfig>;
}

// 干预策略
class InterventionStrategy {
  // 检测干预时机
  async detectInterventionPoint(context: PipelineContext): Promise<InterventionPoint | null>;

  // 生成干预建议
  async generateIntervention(point: InterventionPoint): Promise<Intervention>;

  // 决策是否展示
  shouldShowIntervention(intervention: Intervention): boolean;
}
```
