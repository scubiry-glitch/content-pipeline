## 1. 需求价值（承接 WHY.md）

### 1.1 与 WHY.md 的关联

根据 [WHY.md](../WHY.md)，项目核心目标是建立**标准化、可复用的内容生产流水线**，实现高自动化和稳定产出。

**v4.1 解决的关键痛点**：

| WHY.md 痛点 | v4.1 解决方案 | 价值量化 |
|------------|--------------|---------|
| "产出效率极低（重复劳动）" | 条件触发器自动流转 | 人工干预减少 **70%** |
| "无法规模化" | 智能路由自动分配任务 | 并行处理能力提升 **3倍** |
| "缺乏有效的生产工具链" | 动态任务调度 | 资源利用率提升 **50%** |

### 1.2 编排场景示例

**场景1：低质量自动退回**
```
触发条件: quality_score < 60
自动动作: 退回 Stage 2 补充研究
通知: "内容质量不足，已自动退回研究阶段"
```

**场景2：热点加速通道**
```
触发条件: hot_score > 90
自动动作: 跳过竞品分析步骤
效果: 热点内容提前2小时发布
```

**场景3：蓝军智能匹配**
```
触发条件: content_type = "finance"
自动动作: 分配给金融专家蓝军
效果: 专业内容得到专业评审
```

## 2. 功能清单

### 2.1 条件触发器

**规则配置**:
```yaml
rules:
  - name: "低质量退回研究"
    condition: "quality_score < 60"
    action: "back_to_stage"
    target: 2
    notify: true

  - name: "热点加速发布"
    condition: "hot_score > 90"
    action: "skip_step"
    target: "competitor_analysis"

  - name: "极端情绪预警"
    condition: "sentiment.level == 'extreme_greed'"
    action: "add_warning"
    message: "市场过度乐观，建议增加风险提示"

  - name: "长文自动分段"
    condition: "word_count > 5000"
    action: "split_output"
    target: "series"

  - name: "合规不通过拦截"
    condition: "compliance.score < 70"
    action: "block_and_notify"
    target: "author"
```

**条件类型**:
| 类型 | 说明 | 示例 |
|------|------|------|
| 数值比较 | > < = >= <= | quality_score > 80 |
| 字符串匹配 | == != contains | topic == "AI" |
| 逻辑组合 | AND OR NOT | score > 80 AND hot > 60 |
| 时间条件 | before after | hour < 9 |

### 2.2 智能路由

**路由策略**:
```
内容类型 → 专家匹配:
- 金融 → 金融蓝军专家
- 科技 → 技术蓝军专家
- 政策 → 政策分析师
- 综合 → 组合专家团队

数据完整性 → 处理分支:
- 数据充足 → 进入写作
- 数据不足 → 触发补充研究
- 数据冲突 → 人工确认

平台特性 → 输出策略:
- 公众号 → 长文策略
- 即刻 → 短平快策略
- 知乎 → 深度策略
```

### 2.3 动态任务调度

**优先级队列**:
```
优先级策略:
1. hot_first: 热点内容优先
2. fifo: 先到先处理
3. deadline: 截止时间优先
4. custom: 自定义权重

权重公式:
priority = hot_score * 0.4 + deadline_urgency * 0.3 + manual_boost * 0.3
```

**资源冲突检测**:
- 同一蓝军同时只能处理1个任务
- LLM API 调用频率限制
- 数据库连接池管理

**超时处理**:
```
Stage 1 (研究): 超时 4小时 → 提醒 → 超时 8小时 → 升级
Stage 2 (蓝军): 超时 2小时 → 提醒 → 超时 4小时 → 重新分配
Stage 3 (写作): 超时 1小时 → 自动保存 → 超时 2小时 → 提醒
```

### 4.2 核心模块设计

```typescript
// 编排引擎
class OrchestratorEngine {
  // 注册规则
  async registerRule(rule: OrchestratorRule): Promise<void>;

  // 评估条件
  async evaluateCondition(
    condition: string,
    context: PipelineContext
  ): Promise<boolean>;

  // 执行动作
  async executeAction(
    action: string,
    target: string,
    context: PipelineContext
  ): Promise<ActionResult>;

  // 监听事件
  on(event: string, handler: EventHandler): void;
}

// 任务调度器
class TaskScheduler {
  // 添加任务到队列
  async enqueue(task: Task, priority: number): Promise<void>;

  // 分配任务
  async assignTask(taskId: string, assignee: string): Promise<void>;

  // 处理超时
  async handleTimeout(taskId: string): Promise<void>;

  // 获取队列状态
  async getQueueStatus(): Promise<QueueStatus>;
}

// 智能路由
class SmartRouter {
  // 匹配专家
  matchExpert(contentType: string, experts: Expert[]): Expert | null;

  // 匹配策略
  matchStrategy(platform: string, content: Content): OutputStrategy;

  // 计算优先级
  calculatePriority(task: Task): number;
}

// 工作流状态机
class WorkflowStateMachine {
  // 当前状态
  currentState: Stage;

  // 状态流转
  async transition(event: string, data: any): Promise<Stage>;

  // 获取可用动作
  getAvailableActions(): string[];
}
```
