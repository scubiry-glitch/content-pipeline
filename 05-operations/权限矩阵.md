## 1. 用户角色

### 1.1 角色定义

| 角色 | 描述 | 用户 |
|------|------|------|
| **管理员** | 系统全权管理 | 内容创作者（本人） |
| **蓝军** | 内容评审专家 | AI + 外部专家 |
| **访客** | 只读访问 | 无（当前版本不支持） |

### 1.2 角色说明

当前版本为**单人使用**系统，所有操作均由管理员执行。
蓝军评审由AI自动完成，无需额外用户权限。

## 3. 权限实现

### 3.1 当前版本

**无需权限控制** - 单人系统，所有功能对管理员开放。

### 3.2 未来扩展

如需支持多用户，建议采用：

```
┌─────────────────────────────────────────────────────────────┐
│                      权限控制架构                            │
├─────────────┬────────────────────────────────────────────────┤
│  认证层     │ JWT Token / API Key                            │
├─────────────┼────────────────────────────────────────────────┤
│  授权层     │ RBAC (Role-Based Access Control)               │
├─────────────┼────────────────────────────────────────────────┤
│  数据层     │ Row-Level Security (PostgreSQL RLS)            │
└─────────────┴────────────────────────────────────────────────┘
```

## 4. 安全策略

### 4.1 API 密钥管理

| 密钥类型 | 存储位置 | 访问控制 |
|---------|---------|---------|
| Claude API Key | `.env` 文件 | 仅管理员 |
| Database URL | `.env` 文件 | 仅管理员 |
| 第三方平台 Token | `.env` 文件 | 仅管理员 |

### 4.2 环境隔离

| 环境 | 用途 | 数据隔离 |
|------|------|---------|
