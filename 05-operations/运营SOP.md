## 1. 内容生产流程

### 1.1 标准流程图

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│   选题      │───▶│   研究      │───▶│   写作      │───▶│   发布      │
│  (周一)     │    │ (周一-周二) │    │ (周三-周四) │    │  (周五)     │
└─────────────┘    └─────────────┘    └──────┬──────┘    └─────────────┘
                                             │
                                             ▼
                                      ┌─────────────┐
                                      │  蓝军评审   │
                                      │ (周四晚)    │
                                      └─────────────┘
```

**双周更节奏**:
- 第1周: 选题 → 研究 → 写作 → 评审 → 发布
- 第2周: 数据监控 → 用户反馈 → 下一期准备

## 2. 各环节 SOP

### 2.1 选题环节 (周一)

**负责人**: 内容创作者
**工具**: 系统选题模块
**耗时**: 1-2小时

**步骤**:
1. 登录系统查看「推荐选题」
   - 热点追踪推荐
   - 用户反馈汇总
   - 历史研报关联推荐
2. 评估选题价值
   - 观点独特性
   - 受众需求度
   - 数据可支撑性
3. 确定选题并创建
4. 设定发布目标日期

**检查清单**:
- [ ] 选题与242份研报无重复
- [ ] 有明确的核心观点
- [ ] 有可获取的数据支撑

---

### 2.2 研究环节 (周一-周二)

**负责人**: 系统自动 + 人工校验
**工具**: 系统研究模块
**耗时**: 4-6小时 (人工校验占50%)

**步骤**:
1. 系统自动执行研究
   ```bash
   POST /api/topics/{topicId}/research
   ```
2. 等待研究完成 (10-30分钟)
3. 查看「研究摘要」和「相关研报」
4. 人工补充关键数据
5. 确认研究完整性

**检查清单**:
- [ ] 研究摘要覆盖核心观点
- [ ] 至少3份相关研报被引用
- [ ] 关键数据已核实

---

### 2.3 写作环节 (周三-周四)

**负责人**: 系统自动 + 人工编辑
**工具**: 系统写作模块
**耗时**: 3-4小时

**步骤**:
1. 生成大纲
   ```bash
   POST /api/topics/{topicId}/outline
   ```
2. 审核大纲结构 (30分钟)
3. 生成内容初稿
   ```bash
   POST /api/topics/{topicId}/content
   Body: { "model": "claude-sonnet-4-6" }
   ```
4. 人工编辑初稿 (1-2小时)
5. 提交蓝军评审
   ```bash
   POST /api/content/{contentId}/review
   ```

**检查清单**:
- [ ] 大纲逻辑清晰
- [ ] 内容数据准确
- [ ] 语言风格统一

---

### 2.4 蓝军评审 (周四晚)

**负责人**: AI蓝军 + 人工终审
**工具**: 系统评审模块
**耗时**: 1小时

**评审维度**:
| 维度 | 权重 | 通过标准 |
|------|------|---------|
| 逻辑性 | 30% | 论证完整，无逻辑漏洞 |
| 数据准确性 | 30% | 数据来源可靠，计算无误 |
| 观点独特性 | 20% | 有洞察，非泛泛而谈 |
| 可读性 | 20% | 表达清晰，结构合理 |

**步骤**:
1. AI自动评审 (10分钟)
2. 查看评审报告
   - 总分 > 80分: 通过
   - 总分 60-80分: 需修改
   - 总分 < 60分: 重写
3. 人工终审确认
4. 确认发布

**检查清单**:
- [ ] AI评审总分 > 80
- [ ] 无严重问题（红色标记）
- [ ] 人工确认通过

---

### 2.5 发布环节 (周五)

**负责人**: 系统自动
**工具**: 系统发布模块
**耗时**: 30分钟

**步骤**:
1. 选择发布平台
   - 微信公众号（主阵地）
   - 知乎（扩展）
   - 雪球（财经垂直）
2. 执行发布
   ```bash
   POST /api/content/{contentId}/publish
   Body: { "platforms": ["wechat", "zhihu", "xueqiu"] }
   ```
3. 确认发布成功
4. 记录发布链接

**检查清单**:
- [ ] 各平台发布成功
- [ ] 链接可正常访问
- [ ] 排版显示正常

## 3. 监控与反馈

### 3.1 数据监控指标

| 指标 | 监控频率 | 健康标准 |
|------|---------|---------|
| 阅读量 | 每日 | 较上期±20% |
| 点赞/在看 | 每日 | > 阅读量5% |
| 评论数 | 每日 | > 阅读量1% |
| 分享数 | 每日 | > 阅读量0.5% |
| 新增关注 | 每周 | > 50 |

### 3.2 用户反馈收集

**渠道**:
- 评论区关键词提取
- 私信/留言整理
- 社群讨论监控

**处理流程**:
1. 系统自动抓取反馈 (每日)
2. 每周汇总分析
3. 有价值的反馈转化为下期选题

## 4. 异常处理

### 4.1 常见异常

| 异常场景 | 处理方案 |
|---------|---------|
| AI生成内容质量低 | 切换模型(sonnet→opus)或重写提示词 |
| 蓝军评审不通过 | 根据反馈修改，重新提交评审 |
| 发布失败 | 检查平台API状态，手动补发 |
| 系统故障 | 查看日志，联系开发 |

### 4.2 升级机制

```
问题发生
    │
    ▼
┌─────────────────────┐
│ 能否自行解决？       │
└─────────────────────┘
    │           │
   是          否
    │           │
    ▼           ▼
解决问题    联系技术支持
    │           │
    ▼           ▼
记录备案    升级处理
```
