# 内容生产流水线 — 项目说明书

> AI 驱动的产业研究内容自动化生产系统

---

## 一、项目概述

**内容生产流水线**是一套面向专业内容创作者的 AI 驱动全流程自动化生产系统。系统以 TypeScript + Node.js + PostgreSQL 为技术底座，深度集成多家大语言模型（Kimi / Claude / OpenAI），实现了从**选题发现 → 深度研究 → 智能写作 → 专家评审 → 多平台发布**的端到端内容生产闭环。

项目核心理念：**让内容创作者专注于"思考"，而非"操作工具"**。

---

## 二、项目背景与目标

### 2.1 行业痛点

在产业研究与深度内容创作领域，创作者普遍面临三大困境：

| 痛点 | 现状描述 |
|------|---------|
| **知识碎片化** | 大量历史研究报告（如 242 份研报）分散存储，无法高效检索和复用 |
| **生产效率低** | 从一个观点到一篇成稿需经历研究、写作、排版、多态转换等大量重复劳动，耗时数天 |
| **质量难标准化** | 缺乏系统性的内容评审机制，质量依赖个人经验，难以规模化 |

### 2.2 项目目标

| 维度 | 现状 | 目标 |
|------|------|------|
| 质量保障 | 人工把控 | 蓝军评审机制，100% 通过率 |
| 产出节奏 | 不定期 | 双周稳定更新 |
| 单期耗时 | 数天 | < 2 天 |
| 自动化率 | ~0% | > 70% |
| 知识复用 | 低 | 每次自动关联历史研究成果 |

## 三、功能设计

### 3.1 核心生产流水线

```
选题发现 ──→ 深度研究 ──→ 智能写作 ──→ 蓝军评审 ──→ 人工确认 ──→ 多平台发布
  │             │             │             │                          │
  ▼             ▼             ▼             ▼                          ▼
热点追踪      RAG检索      大纲生成     3专家×2轮              多态转换
历史复用      语义搜索      流式写作     迭代优化              数据监控
竞品监控      数据可视化    多轮迭代     质量评分              A/B测试
```

### 3.2 四大核心模块

**模块一：智能选题**
- 热点话题自动追踪（RSS 聚合 + 趋势检测）
- 基于历史研报的智能选题推荐
- 竞品内容策略监控
- 内容差异化角度挖掘

**模块二：深度研究**
- 基于 pgvector 的语义向量检索，对 242+ 份历史研报实现毫秒级关联
- PDF / Word / HTML 多格式文档解析（含 OCR）
- 外部数据源自动抓取与整合
- 研究摘要与数据可视化自动生成

**模块三：智能写作**
- AI 驱动的大纲自动生成与流式初稿写作
- 多 LLM 供应商智能路由（Kimi → Claude → OpenAI 自动降级）
- 基于评审反馈的多轮自动迭代优化
- 事实核查与数据验证层

**模块四：蓝军评审（BlueTeam Review）**
- **三专家并行评审**：挑战者（逻辑/数据验证）、扩展者（跨领域分析）、归纳者（结构优化）
- 2 轮深度评审 + 迭代反馈闭环
- 量化质量评分与决策框架
- 流式反馈，实时可见评审过程

### 3.3 高级特性

**专家库系统（Expert Library v5.0+）**
- 132+ 位领域专家画像，涵盖房地产金融、FinTech、政策研究等多领域
- **认知数字孪生（CDT）建模**：让 AI 以特定专家的思维方式进行分析
- 专家心智模型（EMM）质量门控
- 1v1 专家对话与多轮深度交流

**内容资产库（Content Library v7.0）**
- 四层分级检索架构：
  - L4 — 跨内容推理（知识图谱、观点演化追踪、矛盾检测）
  - L3 — 混合检索（语义 + 关键词 + RRF/LLM 重排序）
  - L2 — 分层加载（80 / 400 / 1000+ token 三级摘要）
  - L1 — 知识整合（事实三元组提取、实体归一化、增量压缩）
- 15 类消费制品输出（价值选题、趋势信号、差异化角度、知识缺口、密集知识卡片等）

**素材智能处理（Assets AI v6.2）**
- AI 批量文档处理流水线
- 自动标签分类与语义嵌入生成
- 素材关联与关系追踪

## 六、核心创新

### 创新一：蓝军评审机制 — AI 内容质量的"红蓝对抗"

区别于传统 AI 写作工具的"一次生成"模式，本系统首创**三专家并行蓝军评审**机制。每篇内容必须经过挑战者、扩展者、归纳者三种角色的 2 轮独立评审，模拟真实编辑部的质量把关流程，实现 AI 内容从"能用"到"可信赖"的质量跃迁。

### 创新二：认知数字孪生（CDT）— 让 AI 像特定专家一样思考

系统内置 132+ 位领域专家的认知画像，不是简单的 prompt 角色扮演，而是通过**专家心智模型（EMM）**建模其分析框架、思维偏好和判断标准。用户可以与特定领域专家进行深度对话，获得个性化的专业分析视角。

### 创新三：四层分级知识检索 — 从"搜索"到"推理"

传统 RAG 系统止步于相似性检索。本系统构建了四层递进的知识检索架构：
- 底层进行知识整合（事实提取、去重、压缩）
- 中层实现分层加载（按信息密度分级）
- 上层采用混合检索（语义 + 关键词 + 智能重排序）
- 顶层支持跨内容推理（观点演化追踪、矛盾检测、知识图谱）

这使系统不仅能"找到相关内容"，更能"理解内容之间的关系"。

### 创新四：多 LLM 智能路由 — 成本与质量的最优平衡

系统实现了多 LLM 供应商（Kimi / Claude / OpenAI）的智能路由与自动降级机制。根据任务类型、成本预算和服务可用性动态选择最优模型，在保证输出质量的同时有效控制 API 成本。

### 创新五：全流程流式处理 — 从等待到实时协作

大纲生成、初稿写作、专家评审等环节均支持流式输出。用户无需等待完整结果，即可实时查看 AI 的思考与写作过程，随时介入调整方向，实现人机实时协作的创作体验。
